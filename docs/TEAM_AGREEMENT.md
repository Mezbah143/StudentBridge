## Team Roles

| Scribe | Records decisions, updates docs, tracks blockers. |

| AI Steward | Tracks AI-assisted work and code ownership. |

## Communication

Primary channels:

- KakaoTalk for quick coordination.
- GitHub issues and pull requests for project evidence.
- In-person or online meetings for weekly planning and demo rehearsal.

Team expectations:

- Respond within 24 hours when possible.
- Ask for help early.
- Notify the team before missing a meeting or deadline.
- Keep blockers visible in GitHub.

## GitHub Workflow

The team follows:

```text
Issue -> Branch -> Pull Request -> Review -> Merge -> Evidence
```

Rules:

- No major work without an issue.
- Use branches and pull requests for reviewable changes.
- Link tests, screenshots, demo clips, or notes.
- Keep sprint packet evidence updated weekly.

## Evidence Rule

The class rule is:

> If it is not linked, it did not happen.

Every student should post 2-3 receipts each week, such as PRs, commits, issues, review comments, screenshots, test results, or documentation updates.

## AI-Assisted Work Rule

AI-assisted work is allowed only when a human teammate can run it, explain it, test it, debug it, and link evidence. See [`AI_CODE_OWNERSHIP_AUDIT.md`](AI_CODE_OWNERSHIP_AUDIT.md).

## Conflict Resolution

1. Discuss the issue respectfully inside the team.
2. Reassign or reduce scope if needed.
3. Record the blocker or decision in GitHub.
4. Ask the instructor for help if the problem continues.

## Updating This Agreement

Update this file when team expectations or role rotations change. Record the update through a commit or pull request.

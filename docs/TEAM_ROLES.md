## Contribution Receipts

Each member should add 2-3 weekly receipts to the sprint packet issue.

Good receipts:

- Pull request link.
- Issue link.
- Commit link.
- Review comment.
- Screenshot of working feature.
- Demo video.
- Documentation update.
- Test result screenshot.

## Team Working Agreement Summary

- Communicate in KakaoTalk/GitHub and respond within 24 hours when possible.
- Use GitHub issues before starting major work.
- Use branches and pull requests for reviewable changes.
- Record blockers early.
- Keep the demo honest: show what works, document what does not.
- Do not count AI-assisted work unless a team member can run, explain, test, and debug it.

# StudentBridge Design Package

Last updated: 2026-05-13

## Purpose

This design package gives the StudentBridge team a clear visual direction for web app mockups, the midterm pitch, the brochure, and QR-code handouts.

StudentBridge should feel like a trusted university support platform: clean, practical, calm, and easy to explain in a capstone demo.

## Design Tools

| Tool | Use |
|---|---|
| Paper.design | Web app UI mockups for homepage, job search, register/login, application, employer post job, and success states. |
| Canva | Pitch deck, brochure, QR handout, and polished presentation visuals. |
| GitHub docs | Permanent written design package and capstone evidence notes. |

Paper.design is not directly connected in this workspace, so use [`PAPER_DESIGN_UI_SPEC.md`](PAPER_DESIGN_UI_SPEC.md) as the source content to build the mockups there.

## Visual Direction

| Attribute | Direction |
|---|---|
| Tone | Trustworthy, student-friendly, simple, professional |
| Layout style | Clean panels, clear forms, readable job cards, direct navigation |
| Visual density | Moderate: enough information for job search, not overloaded |
| Design personality | Practical university support tool with modern web styling |
| Avoid | Overly decorative startup graphics, fake evidence, unrelated technology stack changes |

## Design System

| Token | Value | Use |
|---|---|---|
| Primary | `#2563EB` | Main buttons, links, selected states |
| Secondary | `#14B8A6` | Support accents, success highlights |
| Accent | `#F59E0B` | Important callouts, QR/evidence highlights |
| Background | `#F8FAFC` | Page background |
| Surface | `#FFFFFF` | Cards, forms, content sections |
| Text | `#0F172A` | Main text |
| Muted text | `#64748B` | Helper text, descriptions |
| Border | `#CBD5E1` | Inputs, card borders, dividers |
| Success | `#16A34A` | Confirmation states |
| Error | `#DC2626` | Error messages |

Recommended fonts:

- Primary: Inter
- Fallback: Noto Sans, Arial, sans-serif

## UI Rules

- Use 8px border radius for cards, forms, and buttons.
- Use full-width buttons on mobile.
- Keep job cards scannable: title, company, location, salary, tags, action.
- Keep forms short and readable.
- Use clear labels instead of placeholder-only fields.
- Use evidence placeholders instead of fake demo links or screenshots.
- Keep the project stack visible as HTML/CSS/JavaScript + Java Servlet + Apache Tomcat + MySQL.
